# AI 協作的抽象化範式

> 記錄日期：2026-01-06

## 核心觀點

人類與 AI 溝通的方式正在演進：**透過抽象化介面，而非純自然語言**。

```
傳統程式設計                    AI 時代
─────────────                  ─────────
人類 → 程式語言 → 電腦          人類 → 抽象介面 → AI → 執行
        ↓                              ↓
      精確控制                    意圖傳達 + 能力授權
```

## 對應關係

| 傳統軟體 | AI 協作 | 作用 |
|----------|---------|------|
| API | MCP | 能力邊界（能做什麼） |
| SDK/Library | Tools | 具體實作（怎麼做） |
| 文檔 + Best Practices | Skill | 領域知識（什麼情況用什麼） |
| Config | CLAUDE.md | 上下文約束（專案規範） |

## 深層洞察

### Skill 是封裝好的「判斷力」

Skill 不只是「知識」，它更像是：

- 不是告訴 AI「有這個工具」
- 而是告訴 AI「**在這種情況下，用這種方式，達成這個目標**」

這與「有主見的設計」(Opinionated Design) 原則一致：
> 減少決策點，合理預設值 > 讓 AI 選擇

## AI 與人類的價值分工

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   AI 的價值：快速從 0 → 60 分                           │
│   ├── 基礎產出、樣板代碼、重複性工作                    │
│   └── 「能用」的版本                                    │
│                                                         │
│   人類的價值：從 60 → 100 分                            │
│   ├── 架構決策、品質把關、細節打磨                      │
│   └── 「好用」「優雅」「正確」的版本                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**實務上的分工模式：**

| 角色 | AI | 人類 |
|------|-----|------|
| 定位 | 實作外包 | 設計 + 審核 |
| 類比 | 快速打字的 Junior | 有經驗的 Tech Lead |
| 產出 | 初稿、候選方案 | 最終決策、品質把關 |

> **AI 是放大器，不是替代品。**
> 有能力的人用 AI 更有效率；沒有能力的人用 AI 只會產出更多垃圾。

## 設計原則

| 原則 | 說明 |
|------|------|
| **有主見的設計** | 合理預設值 > 讓 AI 選擇，必填參數 ≤ 2 個 |
| **深且窄** | 專注 10% 高價值任務，不追求功能廣度 |
| **預期失敗** | 95% Agent 在生產環境失敗是常態，設計優雅降級 |
| **增強回饋** | 執行中提醒目標和進度，失敗時說明影響範圍 |

## 未來角色演變

這意味著未來的「程式設計師」可能變成：

1. **介面設計師** - 設計 MCP、定義能力邊界
2. **知識工程師** - 編寫 Skill、封裝領域判斷力
3. **約束架構師** - 設計 CLAUDE.md、建立品質護欄

## 對 Self-Evolving Agent 的啟發

evolve 的設計方向：

1. **PSB System** - Plan-Setup-Build 環境準備
   - 確保 AI 執行前環境就緒
   - 減少執行時的不確定性

2. **PDCA 循環** - 結構化的執行框架
   - 提供明確的執行步驟
   - 減少 AI 的自由發揮空間

3. **Repo-based Memory** - 結構化的經驗儲存
   - 可檢索的經驗格式
   - 減少 AI 重複犯錯
