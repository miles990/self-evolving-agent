# knowledge-acquisition-4c 整合

> 當 evolve 遇到知識缺口時，使用 4C 方法論進行系統性學習

## 定位關係

| 面向 | evolve | knowledge-acquisition-4c |
|------|--------|--------------------------|
| **核心目標** | 完成任務並從中學習 | 系統性習得新領域知識 |
| **觸發情境** | 有明確任務要執行 | 需要先學會才能做 |
| **主要循環** | PDCA (Plan-Do-Check-Act) | 4C (Collect-Curate-Connect-Create) |
| **輸出** | 任務成果 + 經驗記錄 | 結構化知識體系 |

**關係定義**：4C 是 evolve 在「知識缺口」情境下的**前置學習引擎**。

## 觸發條件

### 自動觸發 4C

| 條件 | 行為 |
|------|------|
| `overall_confidence < 50%` 且任務需要領域知識 | 先執行 4C 再 PDCA |
| 目標含「學習」「了解」「研究」關鍵詞 | 優先使用 4C |
| Fallback Level 2 且需要系統性學習 | 載入 4C 方法論 |
| 連續 3 次相同類型的知識缺口失敗 | 強制進入 4C |

### 路由決策

| 信號 | 路由 |
|------|------|
| 目標含「學習」「了解」「研究」 | 優先 4C |
| 目標含「建立」「實作」「修復」 | 優先 PDCA |
| 信心度 < 30% 且任務需要領域知識 | 先 4C 再 PDCA |

## 整合點

### 1. 能力評估觸發 4C

在 skill-acquisition.md 的 Fallback 機制中：

```
Level 1: Skill + Memory（正常路徑）
Level 2: 外部知識源
  ├─ context7 查詢文檔
  ├─ Web 搜尋最佳實踐
  └─ 若需要「系統性學習」→ 載入 knowledge-acquisition-4c ✨
Level 3: 結構化降級
Level 4: 誠實失敗
```

### 2. 4C CREATE 對接 Memory

| 4C 產出 | 對應 evolve Memory 位置 |
|---------|------------------------|
| 已驗證知識清單 | `.claude/memory/learnings/` |
| 已排除的錯誤觀念 | `.claude/memory/failures/` |
| 決策框架/心智模型 | `.claude/memory/patterns/` |
| 結構化知識體系 | 可能蒸餾為新 Skill |

### 3. 檢查點對應

| evolve 檢查點 | 4C 對應動作 |
|--------------|------------|
| CP1 (Memory 搜尋) | 檢查是否已有該領域的 4C 學習記錄 |
| CP2 (Build/Test) | CURATE 階段的「實測驗證」 |
| CP4 (涌現檢查) | CREATE 階段的「知識蒸餾」 |

## 使用模式

### 模式 A: evolve 自動觸發

```bash
/evolve 學習並實作區塊鏈智能合約

# evolve 自動偵測:
# - 關鍵詞「學習」→ 載入 knowledge-acquisition-4c
# - 執行 4C → 信心度提升 → 執行 PDCA
```

### 模式 B: 先學後做（分離執行）

```bash
# 第一步: 純學習
/evolve 用 4C 方法學習量子計算基礎 --max-iterations 5

# 第二步: 帶著知識執行
/evolve 建立量子計算模擬器
```

### 模式 C: 遇到知識缺口時

```
evolve 執行中...
  ↓
PDCA Check 失敗: 不知道如何設計量子閘
  ↓
失敗診斷: Type A (知識缺口)
  ↓
自動載入 knowledge-acquisition-4c
執行 4C 針對「量子閘設計」
  ↓
信心度提升後繼續 PDCA
```

## 注意事項

1. **不要跳過 CURATE** - 4C 最重要的階段是驗證，否則會學到錯誤知識
2. **時間分配** - COLLECT 20%, CURATE 30%, CONNECT 30%, CREATE 20%
3. **與 Memory 同步** - 4C 產出必須遵守 CP3.5（創建後立即更新 index.md）
4. **避免過度學習** - 學到能執行任務即可，不需要成為專家
