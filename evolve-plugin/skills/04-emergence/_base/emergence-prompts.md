# 涌現 Prompt 模板

> 各涌現等級的標準提示詞模板

## Level 0: 基礎模式

```
任務：[具體任務描述]

執行要求：
1. 嚴格按照指定步驟執行
2. 不要自行擴展範圍
3. 完成後報告結果

驗收標準：
- [標準 1]
- [標準 2]
```

## Level 1: 探索模式

```
任務：[具體任務描述]

執行要求：
1. 完成主要任務後，探索相關改進機會
2. 如果發現更好的方法，可以嘗試
3. 記錄探索過程中的發現

探索方向（可選）：
- 效能優化機會
- 程式碼品質改進
- 相關功能擴展

驗收標準：
- [主要標準]
- 額外：記錄探索發現
```

## Level 2: 涌現模式

```
任務：[具體任務描述]

執行要求：
1. 完成任務的同時，尋找跨領域連結
2. 主動識別可重用的模式
3. 考慮這個解決方案對其他領域的啟發

涌現檢查點：
□ 這個方案是否適用於其他類似問題？
□ 有沒有發現意外的連結或洞察？
□ 是否值得蒸餾為新的 skill 或 pattern？

發現記錄：
- 若有涌現發現，記錄到 .claude/memory/discoveries/
```

## Level 3: 自主模式

```
目標：[高層次目標描述]

授權範圍：
- 可以自主決定實現路徑
- 可以探索非預期的方向
- 可以提出創新性建議

約束：
- 保持在 [約束範圍] 內
- 重大決策前確認
- 記錄所有重要發現

預期輸出：
- 目標達成報告
- 發現的涌現機會清單
- 建議的後續行動
```

## 涌現觸發詞

在目標描述中使用這些詞彙會提高涌現機率：

| 類型 | 關鍵詞 | 效果 |
|------|--------|------|
| 開放性 | 更好、改進、優化 | 允許自主判斷 |
| 探索性 | 探索、發現、嘗試 | 鼓勵多方向 |
| 連結性 | 整合、連結、組合 | 促進跨域 |
| 自主性 | 自主、自動、智能 | 提高決策權 |
| 創新性 | 創新、突破、非預期 | 鼓勵非常規 |

## 使用範例

### 基礎任務 (Level 0)
```bash
/evolve 修復 UserList 組件的 null pointer 錯誤
```

### 探索任務 (Level 1)
```bash
/evolve 修復 UserList 組件的錯誤，順便看看有沒有其他可以改進的地方 --explore
```

### 涌現任務 (Level 2)
```bash
/evolve 改進這個專案的錯誤處理機制 --emergence --max-iterations 10
```

### 自主任務 (Level 3)
```bash
/evolve 讓這個專案變得更好、更智能 --autonomous --max-iterations 10
```
